# ⭐️ M7-02 · Claude Projects 开发最佳实践

> Claude Projects 是持久化的"项目工作区"，比单次对话强大得多。配置好一个 Project，相当于拥有一个永远记得你项目背景的 AI 同事。

---

## Claude Projects vs 普通对话的区别

| 对比 | 普通对话 | Claude Projects |
|------|---------|----------------|
| **上下文** | 每次对话独立 | 永久保留项目背景 |
| **文档** | 每次重新上传 | 一次上传，永久可用 |
| **指令** | 每次重新说明 | 项目指令长期有效 |
| **记忆** | 无 | 在同一 Project 内累积 |

## 开发者必建的 Projects

### 1. 主力项目 Project（每个项目一个）

**Project 指令模板：**
```
你是这个项目的 AI 开发助手。

技术栈：Go (Beego) 后端 + Flutter App + Vue3 Vben Admin 管理后台
当前正在开发：[当前功能/模块]

工作规范：
- 后端分层：controller → service → model，禁止跨层
- 统一返回：{"code": 200, "msg": "...", "data": {}}
- 不确定的地方先问，不要猜

每次回答直接给代码，不需要解释我已经知道的基础概念。
```

**上传的文档：**
- 项目 README（含架构说明）
- 数据库表结构文档
- API 接口文档
- 错误码列表

---

### 2. Code Review Project

**Project 指令：**
```
你是专业的代码审查专家，重点关注：
1. 安全漏洞（SQL注入、权限缺失、密钥泄露）
2. 性能问题（N+1查询、不必要的循环、内存泄漏）
3. 错误处理（是否所有错误都被正确处理）
4. 代码规范（命名、注释、结构）

每次 review 后给出：
- 🔴 必须修复（安全/功能问题）
- 🟡 建议优化（性能/可维护性）
- 🟢 值得学习（优秀写法）
```

---

### 3. 技术调研 Project

**Project 指令：**
```
你是技术调研助手，帮我评估技术方案。

我们的技术约束：
- 后端：Go，不引入 Java 生态
- 前端：Vue3 / Flutter，不用 React
- 部署：Docker + Kubernetes
- 数据库：MySQL 主 + Redis 缓存

每次给出方案时：
1. 优劣势对比
2. 是否符合我们的技术约束
3. 推荐方案 + 理由
4. 潜在风险
```

---

### 4. 文档写作 Project

**Project 指令：**
```
你是技术文档写作助手。

文档风格要求：
- 简洁直接，避免废话
- 有代码示例
- 中文写作
- 结构清晰（用标题、列表、代码块）

上传的文档：公司文档模板.md
每次写文档都参考这个模板的格式。
```

## 高效使用技巧

### 上传什么文档效果最好

| 文档类型 | 效果 | 建议 |
|---------|------|------|
| **数据库表结构** | ⭐⭐⭐⭐⭐ | 必传，AI 写 SQL 直接准确 |
| **API 文档** | ⭐⭐⭐⭐⭐ | 必传，AI 知道有哪些接口 |
| **错误码表** | ⭐⭐⭐⭐ | 强烈推荐，AI 不会乱用错误码 |
| **业务流程图** | ⭐⭐⭐ | 帮 AI 理解业务逻辑 |
| **代码规范文档** | ⭐⭐⭐⭐ | AI 生成代码风格一致 |

### 对话技巧

```
❌ 低效：帮我写一个用户登录接口
✅ 高效：帮我在用户模块新增登录接口，
         需要支持手机号+密码登录，
         失败超过5次锁定账号1小时，
         参考我们的 BaseController 写法
```

## Claude Projects 的局限

- **不能联网** — 无法获取实时信息（可配合 MCP 解决）
- **文件大小限制** — 单文件上限，超大项目需要分割
- **不能执行代码** — 只能生成，不能运行验证（Cursor/Claude Code 可以）

**补充方案：**
- 实时信息 → 配合 Brave Search MCP 或手动提供
- 代码执行 → 切换到 Claude Code 命令行工具
