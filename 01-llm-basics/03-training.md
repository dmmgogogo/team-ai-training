# M1-03 · 训练过程：预训练、微调与 RLHF

> LLM 是怎么从零"学会"说话的？三个阶段搞清楚。

---

## 训练三部曲

```
阶段一：预训练（Pre-training）
    ↓
阶段二：监督微调（SFT - Supervised Fine-Tuning）
    ↓
阶段三：RLHF（基于人类反馈的强化学习）
```

---

## 阶段一：预训练

### 数据
- 来源：Common Crawl（网页）、书籍、GitHub 代码、Wikipedia、学术论文...
- 规模：数万亿 Token（GPT-4 约用了 13 万亿 Token）
- 质量：要大量清洗过滤，去掉垃圾内容

### 目标
**自监督学习**：给模型看一句话的前半部分，让它预测下一个词。不需要人工标注。

```
输入：["今天", "的", "天气", "是"]
目标：["的",   "天气", "是",  "晴天"]（向后移一位）
```

### 成本
- GPT-4 预训练：估计花费 1 亿美元以上
- 需要数千张 A100/H100 GPU，跑数月
- 一旦训练完，模型就"知道"了大量知识

### 结果
得到一个 **Base Model（基座模型）**，能续写文本，但不会"对话"，更不会"听指令"。

---

## 阶段二：监督微调（SFT）

### 问题
Base Model 只会续写文本，你问它"怎么写 Python"，它可能给你续写一道数学题。

### 解决
人工标注大量"指令-回答"对：

```json
{
  "instruction": "用 Python 写一个冒泡排序",
  "response": "def bubble_sort(arr):\n    ..."
}
```

用这些数据微调模型，让它学会"按指令回答"。

### 特点
- 数据量小得多（几千到几十万条）
- 但质量要求高，不好的数据会损害模型
- 效果显著：模型从"续写机器"变成"指令跟随者"

---

## 阶段三：RLHF（基于人类反馈的强化学习）

### 问题
SFT 之后模型能回答问题，但回答质量参差不齐，甚至可能有害。

### 核心思想

让人类评估模型的回答，训练一个**奖励模型（Reward Model）**，再用强化学习优化 LLM。

### 流程

```
1. 让模型对同一问题生成多个答案
        ↓
2. 人工排序：答案 A > 答案 C > 答案 B
        ↓
3. 用排序数据训练奖励模型（打分器）
        ↓
4. 用 PPO 强化学习算法，让 LLM 最大化奖励分数
        ↓
5. LLM 学会产出"人类认为更好"的回答
```

### 效果
- 更有帮助（helpful）
- 更诚实（honest）
- 更无害（harmless）
- Anthropic 称之为 **HHH 原则**

---

## 新进展：DPO 和 GRPO

RLHF 训练复杂、不稳定。近年出现了简化方案：

| 方法 | 特点 |
|------|------|
| **DPO**（Direct Preference Optimization） | 不需要单独训练奖励模型，直接用偏好对训练 |
| **GRPO**（Group Relative Policy Optimization） | DeepSeek-R1 使用，用于训练推理能力 |

---

## 推理模型（o1 / R1 / Claude Extended Thinking）

2024 年后出现的新范式：让模型在回答前先做"内部推理"（Chain of Thought），再给出答案。

```
用户: 9.9 和 9.11 哪个大？
模型内部思考:（看小数点后面，9后面是11，比9大？不，这是小数...）
模型输出: 9.9 更大
```

代表产品：
- OpenAI o1 / o3
- DeepSeek R1
- Claude 3.7（Extended Thinking）

## 关键结论

1. **预训练给知识**，微调给"行为"
2. RLHF 让模型"懂人话、说人话"
3. 训练一次成本极高，推理（用）成本相对低
4. 推理模型通过"慢思考"解决复杂问题
