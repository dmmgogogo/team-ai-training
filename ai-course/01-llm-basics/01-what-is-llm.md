# M1-01 · 什么是大语言模型（LLM）

> 5 分钟读懂 LLM 是什么、能做什么、不能做什么。

---

## 一句话定义

**大语言模型（Large Language Model，LLM）** 是一种在海量文本上训练出来的 AI 模型，能够理解和生成自然语言。

---

## 它是怎么来的？

想象你读了互联网上几乎所有的文章、书籍、代码、论坛帖子——大概是 **数万亿个词**。读完之后，你的大脑形成了对语言模式的深刻理解：什么词后面跟什么词，什么问题通常有什么答案，代码应该怎么写。

LLM 做的就是这件事，但规模大得多，参数量动辄数百亿甚至数千亿。

---

## 它的本质：预测下一个 Token

LLM 的核心任务其实很简单：

```
给定一段文字，预测下一个词（Token）是什么
```

例如：
- 输入："今天天气很"
- 预测：好 / 热 / 冷 / 差 / 晴...（每个词都有概率）

通过一遍遍预测，就能生成连贯的文章、代码、对话。

---

## 能做什么

| 能力 | 例子 |
|------|------|
| 📝 文本生成 | 写文章、邮件、报告 |
| 💬 对话理解 | 客服、问答、聊天 |
| 🧠 推理分析 | 数学题、逻辑推断 |
| 💻 代码编写 | 生成、解释、Debug 代码 |
| 🌐 翻译 | 多语言互译 |
| 📊 摘要提炼 | 长文总结、要点提取 |
| 🔧 格式转换 | JSON/Markdown/表格互转 |

---

## 不能做什么（常见误区）

| 误区 | 真相 |
|------|------|
| ❌ LLM 知道"最新"信息 | 有训练截止日期，不实时联网（除非有搜索插件）|
| ❌ LLM 永远正确 | 会"幻觉"，即自信地说错误的事 |
| ❌ LLM 能记住你 | 默认没有持久记忆，每次对话独立 |
| ❌ LLM 在"思考" | 它在做概率预测，不是真正的推理（虽然很像）|
| ❌ LLM 理解世界 | 它理解文字模式，不一定理解世界本质 |

---

## 参数量代表什么？

常听到"70B 模型"、"千亿参数"，参数就是模型中可学习的权重数量：

- 参数越多 → 能记住更多知识，理解更复杂的问题
- 参数越多 → 运行需要的硬件越贵，速度也更慢
- 不是越大越好，小模型经过优化也能打过大模型

常见量级：
- **7B / 8B**：可以本地跑，适合轻量任务
- **70B**：需要多张 GPU，质量较好
- **405B+**：需要专业集群，顶级质量（如 GPT-4、Claude Opus）

---

## 关键词速查

| 术语 | 解释 |
|------|------|
| **Token** | 模型处理的最小单位，约等于 0.75 个英文单词或半个中文字 |
| **Context Window** | 模型一次能"看到"的最大 Token 数，如 128K |
| **Temperature** | 控制输出随机性，0 = 确定性高，1+ = 更有创意 |
| **Hallucination（幻觉）** | 模型自信地输出错误信息 |
| **Inference（推理/推断）** | 使用训练好的模型生成答案的过程 |
