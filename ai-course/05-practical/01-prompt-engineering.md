# M5-01 · Prompt 工程入门

> 同样的 AI，会写 Prompt 的人能发挥出 10 倍效果。

---

## 什么是 Prompt 工程？

**Prompt Engineering** 是通过设计和优化输入文本，让 LLM 输出你想要的结果的技术。

类比：跟 AI 打交道就像指导一个聪明但需要明确指令的实习生。指令越清晰，结果越好。

---

## 基础原则

### 原则一：明确具体

❌ 差的 Prompt：
```
帮我写代码
```

✅ 好的 Prompt：
```
用 Python 写一个函数，接收一个用户 ID（整数），
从 SQLite 数据库查询该用户的订单列表，
返回格式为包含 order_id、amount、status 字段的字典列表。
数据库路径从环境变量 DB_PATH 读取。
```

---

### 原则二：给出背景（Context）

❌ 差：
```
这段代码有什么问题？
```

✅ 好：
```
我在用 FastAPI + SQLAlchemy 开发用户认证系统。
下面这段代码在并发请求下偶尔出现 session 错误，
帮我分析原因并修复：

[代码]
```

---

### 原则三：指定输出格式

```
分析以下 5 个竞品，用 Markdown 表格输出，
列包含：产品名、核心功能、价格、优势、劣势。
```

---

### 原则四：角色设定（Role Prompting）

```
你是一个有 10 年经验的 Python 高级工程师，
专注于高性能后端系统。
请帮我 review 以下代码，重点关注性能和安全问题：

[代码]
```

---

### 原则五：少样本示例（Few-Shot）

```
将以下 JSON 字段名从驼峰转为下划线命名：

示例：
输入：{"userId": 1, "firstName": "John"}
输出：{"user_id": 1, "first_name": "John"}

现在处理：
{"orderStatus": "paid", "createdAt": "2025-01-01"}
```

## 进阶技巧

### Chain of Thought（思维链）

让模型一步步思考，而不是直接给答案：

```
[直接问]
9.9 和 9.11 哪个大？

[加上 CoT]
9.9 和 9.11 哪个大？请一步步分析，然后给出答案。
```

对于复杂问题，加上"Let's think step by step"显著提升准确率。

---

### 结构化 System Prompt

开发 AI 应用时，用 System Prompt 设定规则：

```
System Prompt 模板：

# 角色
你是 XX 公司的技术助手，专门帮助开发团队解决技术问题。

# 能力范围
- 代码 Debug 和优化
- 技术方案设计
- 代码 Review

# 限制
- 不讨论业务决策和公司战略
- 涉及敏感数据时提示脱敏

# 输出规范
- 代码用代码块包裹并注明语言
- 分析结论用要点列表
- 不确定的内容明确说明
```

---

### XML 标签结构化输入

Claude 特别擅长处理 XML 格式：

```
<context>
这是一个电商系统，使用 Django + PostgreSQL
</context>

<code>
def get_user_orders(user_id):
    return Order.objects.filter(user_id=user_id)
</code>

<task>
添加分页功能，每页 20 条，并缓存结果 5 分钟
</task>
```

---

### 让模型自我验证

```
帮我设计一个数据库 Schema 用于存储用户订单信息。
设计完之后，检查是否满足：
1. 支持一个用户多个订单
2. 支持订单多个商品
3. 方便统计每月销售额
如有不足，自动修改。
```

## 常用 Prompt 模板

### Debug 模板
```
我遇到了以下错误：
[错误信息]

相关代码：
[代码]

我已经尝试过：
[你试过的方法]

请帮我找出根本原因并给出修复方案。
```

### Code Review 模板
```
请 Review 以下代码，关注：
1. 潜在 Bug
2. 安全漏洞（SQL 注入、XSS 等）
3. 性能问题
4. 代码可读性
5. 是否符合 [你的技术栈] 最佳实践

[代码]
```

### 技术方案设计
```
我需要设计 [功能名称]，需求如下：
- [需求1]
- [需求2]
- [需求3]

技术栈限制：[你的技术栈]
并发量：[预计 QPS]

请给出：
1. 方案设计（架构图用文字描述）
2. 关键技术选型及原因
3. 潜在风险和解决方案
4. 实现步骤
```

## 常见错误

| 错误 | 后果 | 解决 |
|------|------|------|
| Prompt 太模糊 | 输出跑偏 | 加约束和示例 |
| 一次问太多 | 顾此失彼 | 拆成多个 Prompt |
| 没说输出格式 | 格式不一致 | 明确说"用 JSON/表格/列表" |
| 完全信任输出 | 踩坑 | 关键内容必须人工验证 |
| 不给上下文 | 答案不相关 | 提供背景信息 |

---

## 自动评估 Prompt 质量

可以让 AI 帮你改进 Prompt：

```
我有以下 Prompt，但效果不太好：

[你的 Prompt]

期望的输出是 [描述期望]，
实际输出是 [描述实际]。

请帮我改进这个 Prompt。
```
