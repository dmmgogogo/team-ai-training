# 文档维护机制（自动每 3 天）

本仓库已加入“文档自动巡检与刷新”机制，用于降低内容过期风险。

## 机制说明

- 触发频率：每 3 天自动执行一次（GitHub Actions 定时任务）
- 执行脚本：`scripts/refresh_docs.py`
- 脚本职责：
  - 更新首页 `README.md` 的“最近更新”日期
  - 更新关键文档的“最后更新”日期标签（当前覆盖 M2/M3/references）
  - 生成巡检报告 `reports/auto-refresh-report.md`
  - 若有变更自动提交（机器人账号）

## 触发方式

1. 自动触发（cron）
2. 手动触发（Workflow Dispatch）
3. 手动执行脚本（本地或 CI）

## 本地手动执行

```bash
python3 scripts/refresh_docs.py
```

执行后会看到：

- `reports/auto-refresh-report.md` 更新
- 相关文档日期字段更新

## 维护原则（深度更新版）

为降低“数字过期误导”风险，课程文档改为以下策略：

1. **保留长期有效信息**：定位、适用场景、选型方法、官方入口
2. **删除高波动硬编码数据**：固定跑分、固定价格、固定市场份额
3. **提供实时查询入口**：官方定价页、权威榜单、厂商文档
