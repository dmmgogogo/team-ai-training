## 必配：Custom Instructions（自定义指令）

**位置：** 右上角头像 → 自定义 ChatGPT

这是让 ChatGPT 永远记住你的偏好的核心设置，分两部分：

### 第一部分：关于你的背景
```
我是一个后端开发工程师，主要使用 Python 和 Go。
我们公司使用 AWS 作为云服务，PostgreSQL 作为主数据库。
我的英语不是母语，请用中文回答。
我有 5 年开发经验，不需要解释基础概念。
```

### 第二部分：你希望 ChatGPT 如何回应
```
- 代码示例优先，减少文字解释
- 直接给出最佳实践，不需要列出多个选项
- 代码要包含错误处理和类型注解
- 遇到不确定的事情，明确说明而不是猜测
- 回答要简洁，不要废话
- 复杂问题先给结论，再解释原因
```

## 内置工具的正确打开方式

对话框左下角有工具图标，展开后可以选择：

### 联网搜索
```
何时用：
✅ 查最新文档、版本信息
✅ 搜索刚发布的工具或库
✅ 验证时效性信息

何时不用：
❌ 通用编程问题（训练数据已有，搜索反而慢）
❌ 代码调试（不需要实时信息）
```

### 代码解释器（Code Interpreter）
```
最强大的内置工具，能执行 Python 代码。

使用场景：
1. 数据分析
   上传 Excel/CSV → "分析销售趋势，画折线图，找出异常"
   
2. 文件处理
   上传 PDF → "提取所有表格数据，转成 Excel"
   上传图片 → "识别图中的文字内容"
   
3. 数学计算
   "解这道微分方程，给出解析解和数值解，画出图像"
   
4. 代码运行验证
   "运行这段 Python，告诉我输出结果"
   "这个正则表达式匹配 '2026-03-14' 吗？测试一下"
```

### 图像生成（DALL-E）
```
触发方式：直接描述你想要的图

有效 Prompt 结构：
"[风格] + [内容描述] + [细节] + [构图]"

例：
"扁平插画风格，一个开发者坐在电脑前，
屏幕显示代码，背景有 AI 机器人在协作，
暖色调，简洁现代"
```

## GPT 功能（自定义 AI 助手）

**位置：** 左边栏 → 探索 GPT / GPT 商店

你可以创建专属 GPT，也可以使用别人分享的：

### 创建团队专属 GPT

**位置：** 右上角 → 我的 GPT → 创建

```
名称：[公司名] 技术助手

指令（System Prompt）：
你是 XX 公司的内部技术助手。
你熟悉我们的技术栈：Python/FastAPI/PostgreSQL/AWS。
你了解我们的代码规范（参见上传的文档）。
回答时优先考虑我们的实际使用场景。
遇到不确定的内容，明确说明并建议查阅内部文档。

上传知识库：
- 公司编码规范.pdf
- API 设计指南.md
- 常见问题 FAQ.md
```

创建好后可以分享给团队成员使用。

## Projects（项目上下文）

**位置：** 左边栏 → 项目 → 新建项目

Projects 让 ChatGPT 记住特定项目的上下文：

```
创建项目：后端开发项目

添加项目指令：
"这个项目是我们的电商后端，使用 FastAPI + PostgreSQL。
以下是项目的数据库 Schema：[粘贴 Schema]
以下是 API 规范文档：[粘贴文档]
本项目所有对话都要基于这个上下文。"

上传文件：
- 项目 README.md
- 数据库设计文档
- API 接口文档
```

之后在这个项目里的所有对话，ChatGPT 都会记住这些背景。

## o3 / o3-pro：何时用推理模型

o3 比 GPT-4o 慢，但在以下场景**显著更好**：

```
✅ 适合用 o3：
- 复杂算法设计（动态规划、图算法）
- 数学证明或推导
- 多步骤逻辑推理
- 代码 bug 的深层原因分析
- 架构方案对比和决策

❌ 不需要 o3（用 GPT-4o 即可）：
- 写一个简单的 CRUD 接口
- 解释一段代码
- 翻译文本
- 日常问答
```

**使用方式：** 对话框左下角模型选择 → o3 / o4-mini

## 记忆功能

**位置：** 设置 → 个性化 → 记忆

开启后 ChatGPT 会自动记住你说过的重要信息。

你也可以主动告诉它：
```
"记住：我们公司所有项目都用 poetry 管理依赖"
"记住：我更喜欢 PostgreSQL 而不是 MySQL"
"记住：代码示例用中文注释"
```

查看记忆内容：设置 → 个性化 → 管理记忆

## 高效对话技巧

### 分步拆解复杂任务
```
❌ "帮我设计一个完整的微服务架构"

✅ 步骤一："列出设计微服务系统需要考虑的维度"
✅ 步骤二："针对服务拆分这个维度，给出具体方案"
✅ 步骤三："这个方案的服务间通信该如何设计？"
```

### 要求输出格式
```
"用表格对比 Redis、Memcached、本地缓存三种方案"

"用 Mermaid 语法画出这个系统的流程图"

"给我一个 JSON 格式的配置模板"
```

### 连续迭代
```
→ "写一个用户注册接口"
→ "加上邮件验证功能"
→ "把错误信息改成中文"
→ "添加请求频率限制，同一 IP 1分钟内只能注册3次"
```
